Overview

Funny API names over normal browser behavior.

MeetingSpeak is a one-file JavaScript runtime. It does not compile anything, invent a renderer, or require a build pipeline. It wraps common browser APIs in intentionally ridiculous business jargon.

What it is

A thin runtime layer over selectors, events, storage, fetch, timers, and ARIA helpers.

What it is not

A parser, framework, component system, template engine, or reactive state library.

Runtime include

Load one script tag and start using globals.

<script src="MeetingSpeak.js"></script>

Why

A runtime joke ships better than a fake language demo.

The concept works because the code actually runs in a browser. You can inspect it, debug it with normal tooling, and still keep the joke intact because the weirdness lives in the API names rather than in magical behavior.

Quick start

Open a plain HTML file and call the runtime directly.

MeetingSpeak

<div id="status"></div>
<button id="save">Save</button>
<script src="MeetingSpeak.js"></script>
<script>
  BROADCAST('#status', 'Ready to align.')
  ACTION_ITEM('#save', () => {
    BROADCAST('#status', 'Draft saved.')
  })
</script>

Equivalent browser JavaScript

const status = document.querySelector('#status')
const save = document.querySelector('#save')

status.textContent = 'Ready to align.'

save.addEventListener('click', () => {
  status.textContent = 'Draft saved.'
})

Installation

Use a script tag. That is the whole setup story.

./MeetingSpeak.js
./docs/index.html
./demo/index.html

Functions are exposed globally by design, but the namespace is available too: MeetingSpeak.BROADCAST('#status', 'Ready').

Core concepts

Selector-first, callback-last, browser-native semantics.

Thin wrappers

Most functions are direct wrappers over `querySelector`, `fetch`, timers, or localStorage.

Readable failure

Missing selectors and bad inputs throw explicit runtime errors instead of failing silently.

Plain state

`SOURCE_OF_TRUTH` stores values in a normal object. Nothing rerenders until you do it yourself.

No compiler

The current project is JavaScript with themed function names, not a standalone language parser.

API reference

Canonical v1 surface

Showing all groups.

No API groups match that filter.

DOM selection

  • FIND_STAKEHOLDER(selector) returns the first match.
  • FIND_STAKEHOLDERS(selector) returns all matches.
  • IS_ON_THE_CALENDAR(selector) checks existence.
  • READ_THE_ROOM(selector) reads textContent.
  • TAKE_THE_TEMPERATURE(selector) reads value when available.

DOM updates

  • ORG_UPDATE(selector, html) sets innerHTML.
  • BROADCAST(selector, text) sets text.
  • SOCIALIZE(selector, html) appends HTML.
  • SUNSET(selector) removes the matched element.
  • ABOVE_THE_FOLD(selector) scrolls an element into view.

Styles and classes

  • ALIGN_ON(selector, className) adds a class.
  • DEPRIORITIZE(selector, className) removes a class.
  • BRAND_REFRESH(selector, styles) applies inline styles.
  • TAKE_COVER(selector) hides an element.
  • VISIBILITY_INTO(selector) shows an element.

Forms and attributes

  • CAPTURE_INPUT(selector) returns value.
  • PREFILL(selector, value) sets value.
  • GET_BUY_IN(selector) reads checked state.
  • LOCK_IT(selector) and UNBLOCK(selector) disable or enable controls.
  • TAG_FOR_FOLLOWUP(selector, key, value) and PULL_METADATA(selector, key) wrap dataset.

Events

  • LISTEN_IN(selector, eventName, handler) attaches a listener.
  • ACTION_ITEM(selector, handler) is the click shorthand.
  • FOLLOW_UP(selector, handler) listens to input and change.
  • ESCALATE(eventName, detail) dispatches a document-level custom event.

State and utility

  • SOURCE_OF_TRUTH(name, value) stores a value in the runtime state object.
  • ALIGN(name, value) updates the same store.
  • SYNERGIZE(a, b) adds numbers or concatenates strings.
  • RIGHTSIZE(value) normalizes common string inputs.

Collections

  • PIPELINE(items, fn) maps items.
  • FILTER_FOR_FIT(items, fn) filters items.
  • NET_NET(items, fn, initial) reduces items.
  • SORT_THE_DECK(items, fn) sorts a shallow copy.

Async and scheduling

  • CIRCLE_BACK_IN(ms, fn) wraps setTimeout.
  • RECURRING_SYNC(ms, fn) wraps setInterval.
  • CANCEL_THE_SERIES(id) clears timeouts or intervals.
  • BY_CLOSE_OF_BUSINESS(fn) queues work soon without blocking.
  • TIMEBOX(ms, promise) rejects if the promise takes too long.

Network

  • OUTREACH(url, options) is a thin fetch wrapper.
  • PULL_NUMBERS(url) fetches JSON.
  • SUBMIT_DECK(url, data) posts JSON.

Storage

  • INSTITUTIONAL_KNOWLEDGE(key, value) serializes to localStorage.
  • LOOK_IT_UP(key) reads and parses.
  • MEMORY_HOLE(key) deletes the key.

Logging and debugging

  • EXECUTIVE_SUMMARY(...args) logs.
  • FLAG_FOR_REVIEW(...args) warns.
  • FIVE_ALARM_FIRE(...args) errors.
  • SANITY_CHECK(value, msg) throws on falsy values.

Accessibility

  • FOCUS_THE_ROOM(selector) moves focus.
  • INCLUSIVE_ALIGNMENT(selector, label) sets an ARIA label.
  • GIVE_IT_A_VOICE(selector, text) writes to a live region.

Philosophy

The joke is in the names, not in hidden semantics.

MeetingSpeak stays readable because it does not try to outsmart the platform. Underneath the themed function names, browser behavior stays ordinary and debuggable.

Roadmap

Runtime first, parser later if it is still funny.

  • v1 remains a browser runtime DSL.
  • v1.5 could add more debug affordances and a few helper functions.
  • v2 could explore a parser and transpiler for standalone MeetingSpeak source.

Limitations

Intentional boundaries keep the project understandable.

  • No build tooling or module packaging is part of this project today.
  • No reactive rendering system or template engine exists in v1.
  • Network helpers are convenience wrappers, not a full client abstraction.
  • This is a browser runtime DSL, not a standalone language.
