Overview

v2 adds just enough rendering, state, feedback, and event ergonomics to build richer static apps.

MeetingSpeak still ships as a single browser script. v2 keeps the v1 wrappers intact, then layers on list rendering, state subscriptions, loading helpers, feedback banners, keyboard shortcuts, and optional tone-mode comedy.

Still browser-first

No build step, no dependencies, no compiler, no reactive diffing engine.

More capable than v1

v2 adds rendering helpers, subscriptions, feedback helpers, storage metadata, and loading states.

Runtime include

Load one file and use either globals or the namespace.

<script src="MeetingSpeak.js"></script>

What's new in v2

The runtime got richer without becoming a tiny fake framework.

Rendering helpers

SPIN_UP, REORG, RUN_IT_UP_THE_FLAGPOLE, and LOW_HANGING_FRUIT reduce common DOM boilerplate.

State subscriptions

OPEN_LOOP, CLOSE_THE_LOOP, LISTEN_FOR_ALIGNMENT, and TRUE_UP make small apps much easier to wire.

Feedback and polish

WIN_THE_ROOM, TAKE_THE_L, PULSE_CHECK, MAKE_IT_POP, and QUIET_QUITTING support real UI feedback.

Sharper comedy

Tone modes, structured diagnostics, and a small curated alias set add more identity without turning the API into noise.

Quick start

Open a plain HTML file and render from state.

MeetingSpeak v2

<ul id="queue"></ul>
<script src="MeetingSpeak.js"></script>
<script>
  OPEN_LOOP('queue', ['Columbo', 'Party Down'])

  function renderQueue(items) {
    RUN_IT_UP_THE_FLAGPOLE('#queue', items, (title) => `<li>${title}</li>`)
  }

  LISTEN_FOR_ALIGNMENT('queue', renderQueue)
  renderQueue(SINGLE_SOURCE_OF_TRUTH('queue'))
</script>

Equivalent browser JavaScript

const queue = ['Columbo', 'Party Down']
const queueEl = document.querySelector('#queue')

function renderQueue(items) {
  queueEl.innerHTML = items.map((title) => `<li>${title}</li>`).join('')
}

renderQueue(queue)

Globals are exposed by design, but the namespace is still available: MeetingSpeak.RUN_IT_UP_THE_FLAGPOLE(...).

      Compatibility

      v2 is backward compatible with the existing v1 public API.

      • All v1 canonical functions remain available globally and under window.MeetingSpeak.
      • SOURCE_OF_TRUTH and ALIGN still behave as direct state assignments.
      • v2 layers new helpers on top instead of renaming or replacing the v1 surface.
      • The project remains a browser runtime DSL, not a standalone language, parser, or framework.

      Tone modes

      Comedy is optional and mostly affects log and error copy.

      Professional keeps the voice mostly restrained.

      Boundaries

      The runtime is still intentionally small and inspectable.

      • No virtual DOM, reactive diffing engine, or component lifecycle system.
      • No build tooling or compilation is required to use the project.
      • No parser, custom source format, or faux programming language sits behind the API.
      • The wrappers stay thin so the browser behavior is easy to inspect directly.
